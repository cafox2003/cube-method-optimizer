# Context: Custom Method Solver Project

## Overview

This project is a **generic Rubik's Cube method solver framework**.

Its core goal is:

> To solve a cube using *custom-defined solving methods*, expressed through a DSL, rather than being hardcoded to CFOP, Roux, ZZ, etc.

This is not just a solver — it is a **method execution engine** that orchestrates constrained search through a backend solver, generates data about method performance, and will eventually use ML to discover and evaluate new methods.

---

## System Layers

```
kubesolver (C binary)
    ↓
core/ — models, DSL parser, rotation, cache
    ↓
solver/ — MethodRunner execution engine
    ↓
generation/ — data pipeline (solves, evaluation, method vectors)
    ↓
ml/ — score prediction from method vectors       [planned]
    ↓
discovery/ — mutation-based method search loop   [planned]
```

Each layer treats the one below it as a black box. The ML layer reads from files, never imports from solver. The discovery loop drives the full stack but owns none of the internals.

### Key Design Changes in the Refactor

**`workspace_root` threading**: cache and data paths are no longer hardcoded. Every function that reads or writes files takes a `workspace_root` argument — either `workspace/stable` or `workspace/scratch`. This is what makes the stable/scratch separation work cleanly.

**`bin/` platform resolution**: solver binary path is resolved at runtime via `platform.system()` in both `solver/run.py` and `generation/data_generation.py`. No hardcoded paths.

**Clean import graph** — no circular dependencies:
```
core/  ←  solver/  ←  generation/  ←  discovery/
                   ←  ml/
```
`core` imports nothing from the project. `ml` imports only from `core` (for `method_vector`) and reads from files — never imports `solver` or `generation`.

**`parse_dsl` return value**: the old version returned `(method, runtime_commands)`. The refactored `parse_dsl` returns just `Method` — `runtime_commands` was already being discarded everywhere.

**`test_rotation.py`**: move to `tests/` and update `from rotation import ...` → `from core.rotation import ...`.

## Data Model

### DSL Items
- **`Step`** — a named set of constraints. `step.constraints: list[str]` is mutable and the ML/discovery layer can add, remove, or reorder constraints directly.
- **`Group`** — contains multiple steps with an execution order (`in_order`, `random`, `best`, `worst`, `best_n`, `worst_n`). Groups treat combinatorics as the user's responsibility.
- **`Remove`** — first-class item in `method.items`. Textual removal: removes any constraint line that exactly matches the string. No semantic normalization.
- **`Method`** — top-level container. Holds `items: list`, `rotation_str`, `symmetry_orientations`, `symmetry_depth`.

### Solve Results
- **`StepResult`** — `name: str`, `solution: str` (moves string, `""` if already satisfied)
- **`SolveResult`** — `scramble`, `orientation`, `steps: list[StepResult]`, `failed: bool`
- **`format_solve_result(result)`** — reconstructs human-readable output for CLI use

### Method Vector
`method_vector(method)` in `generation/data_generation.py` produces the flat feature dict used for ML. Key fields: step/group/remove counts, constraint density stats, constraint type breakdown (edge/corner/orientation), symmetry config. `method_name` is included as an identifier — drop it before feeding to a model.

---

## Constraint Model

- Constraints **accumulate** across steps and persist unless explicitly removed
- Removal is **purely textual** — `[REMOVE: add_edge UF]` removes any line matching that exact string
- All preservation is enforced implicitly by accumulated constraints

## Workspace Convention

| Path | Contents | Safe to clear? |
|------|----------|---------------|
| `workspace/stable/` | Hand-curated DSLs, promoted methods, their data | Never |
| `workspace/scratch/` | Discovery-generated DSLs and data | Yes, between runs |

The discovery loop writes everything to `scratch/`. When a method is worth keeping, promote it manually: copy DSL to `stable/dsl/`, alg cache to `stable/algs/`, regenerate data into `stable/data/`.
